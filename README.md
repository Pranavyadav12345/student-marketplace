## 📌 Introduction
The **Student Marketplace Platform** is a web-based application inspired by platforms like NoteKart, designed to enable students to buy, sell, and exchange academic resources such as notes, books, electronics, and study materials.

This system is enhanced with **DevOps practices**, enabling automated deployment, scalability, and continuous integration, making it suitable for real-world production environments.

---

## 🎯 Objectives
- Build a student-centric marketplace platform  
- Enable peer-to-peer exchange of academic resources  
- Implement DevOps pipeline for automation  
- Ensure scalability and high performance  

---

## ❗ Problem Statement
Students face:
- High cost of books and materials  
- Lack of trusted student platforms  
- Difficulty in selling unused items  

Traditional systems lack:
- Automation  
- Continuous deployment  
- Scalability  

---

## 💡 Proposed Solution
The platform provides:

## 🏗️ System Architecture

![Architecture](docs/architecture.png)

### 🔹 Architecture Description
The system follows a **3-tier architecture integrated with DevOps tools**:

- **Frontend:** React.js (User Interface)  
- **Backend:** Node.js + Express (API & logic)  
- **Database:** MongoDB (Data storage)  

DevOps Integration:
- GitHub for version control  
- Jenkins/GitHub Actions for CI/CD  
- Docker for containerization  
- AWS for deployment  
- Nginx for reverse proxy  
- Prometheus & Grafana for monitoring

## 🔄 Workflow Diagram

![Workflow](docs/workflow.png)

### 🔹 Workflow Explanation
1. Developer pushes code to GitHub  
2. CI/CD pipeline is triggered  
3. Application is built and tested  
4. Docker image is created  
5. Application is deployed on AWS  
6. Nginx serves the application  
7. Monitoring tools track performance
